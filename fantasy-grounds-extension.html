Fantasy Grounds Extension

I often try my hand at diving into the source code of games I play to make my own modifications. Most of the time I don't make anything significant and just learn about the inner workings of the program, but this extension is an example of a time where I did more.

Fantasy Grounds is a Virtual Tabletop program, or VTT, which enables users to play conventionally in-person roleplaying games together from anywhere in the world. I've been using it for a few years to play Dungeons and Dragons, and when I wanted to modify the built-in ruleset for the game I had the idea of creating my own extension for Fantasy Grounds.

The extension, which I named Armor Hit Brackets, can be found on my GitHub page. If you're not familiar with Dungeons and Dragons, then my explanation of what I've changed in game terms won't make much sense, but what I've essentially done is interrupted one of the pipelines set up to automate the 'attack' system of the game. In Dungeons and Dragons, when a player wishes to attack something, perhaps a monster, then they must roll a 20-sided die (or d20) and the result determines whether the attack hits or misses. This outcome can be changed based on the attack modifier of the player's character, which gives a bonus to the roll, and the defensive value of the target of the attack, known as AC. If the result of the roll is less than the target's AC, the attack misses.

I experimented with a more 'realistic' version of this attacking system, which introduces the concept of a 'glancing' hit/miss. This makes the average output of an attack, its damage to the target, more consistent by allowing the attack to deal half damage if the result was sufficiently close to the target's AC.

I'm sure you're already very confused as to what any of that means and I don't blame you. What's more important is how I figured out how to implement it to be handled automatically within Fantasy Grounds. This required working out where in the code attack rolls and their results were handled, and then overriding relevant functions to include additional code to handle the calculation of these 'glancing' attacks. To do this, I employed my usual method of scouring the source code of the program and repurposing existing code for my own use.

In short, I eventually succeeded. Although I wasn't very familiar with Lua (the language that Fantasy Grounds is written in) prior to this venture, I managed to make sense of everything to slip my changes into the right parts of the pipeline. Ironically, in the end I chose not to make use of my extension as it further complicated the actual Dungeons and Dragons game mechanics and did more harm than good, but I learned a lot about how Fantasy Grounds works which made it all worthwhile.
